# Contributing to Git-OSINT

First off, thank you for considering contributing to Git-OSINT! It's people like you that make open source software such a great community.

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please ensure the bug was not already reported by searching on GitHub under [Issues](https://github.com/AnkurMacwan/git-osint/issues). If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/AnkurMacwan/git-osint/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample or an executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

If you have an idea for an enhancement, please open an issue to discuss it. This allows us to coordinate our efforts and prevent duplication of work.

1.  **Search for existing suggestions:** Your idea may have already been discussed.
2.  **Open a new issue:** If your idea is new, open an issue. Provide a clear and detailed explanation of the feature you're proposing and why it would be a valuable addition.

### Code Contribution

We welcome pull requests. If you're planning to contribute code, please follow these steps:

1.  **Fork the repository** and create your branch from `main`.
2.  **Set up your development environment.**
3.  **Make your changes.** Ensure your code adheres to the existing style.
4.  **Add or update tests** if your changes require them.
5.  **Ensure all tests pass.**
6.  **Submit a pull request** to the `main` branch of the upstream repository.

Please note that this project has a [Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
